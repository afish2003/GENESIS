## Risks and Fragile Areas

- **Schema discipline**: The pattern of defaulting controller-populated fields works but is fragile. If a new schema is added with a required `agent_id` field, it will hit the same validation crash. Convention: any field the controller sets post-parse MUST have a default.

- **Doctrine revision append strategy**: `doctrine_revision.py` appends revision text to the document rather than applying a true diff. This is functional but will cause doctrine documents to grow linearly. Acceptable for pilot; may need refinement for 100-cycle main study runs.

- **Context window pressure**: The agent system prompt assembles identity + doctrine + last 10 memory entries. Over long runs, this will grow. Token counting is available via tiktoken but not yet used to enforce limits. Monitor context size in pilot runs.

- **Single model for all roles**: All 6 roles (Axiom, Flux, evaluator, memory summarizer, retrieval summarizer, annotation assistant) use the same Ollama model. Evaluator and memory summarizer quality depends on the model's ability to follow structured-output instructions reliably. Qwen2.5:32b handles this well but monitor for schema validation retry rates.

- **Sequential inference**: All model calls are sequential (not parallel). At ~14 calls per cycle, cycle time will be dominated by inference latency. Profile in pilot runs; parallelization is a v2 optimization.
